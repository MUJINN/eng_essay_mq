## 一、设计原则

### 🎯 核心目标

1. **兼容性**：完全兼容现有 AYX 模型的数据结构
2. **扩展性**：新增维度分数和详细评语
3. **一致性**：与 Dianxin/Dify 模型保持字段一致
4. **可用性**：前端可直接使用，无需额外处理

### 📊 设计对比

| 维度 | AYX 模型 | 大模型（目标） |
|------|---------|--------------|
| **分数** | ✅ score（0-1） | ✅ score（0-1） |
| **语法错误** | ✅ error_set | ✅ error_set（增强） |
| **亮点表达** | ✅ highlight | ✅ highlight（增强） |
| **维度分数** | ❌ 无 | ✅ 4个维度（5分制） |
| **总评语** | ❌ 无 | ✅ 详细评语 |
| **改进建议** | ❌ 无 | ✅ 具体建议 |
| **错误纠正** | ❌ 无 | ✅ 原文+修改+原因 |
| **优秀表达** | ❌ 无 | ✅ 句子+点评 |

## 三、字段详细说明

### 🔷 基础分数字段

#### 3.1 percentageScore（百分比分数）

```json
"percentageScore": 75.5
```

- **类型**: `float`
- **范围**: 0-100
- **说明**: 百分比形式的得分率
- **计算**: `(实际得分 / 满分) × 100`
- **示例**:
  - 满分15分，得11.3分 → percentageScore = 75.5
  - 满分25分，得20分 → percentageScore = 80.0

#### 3.2 totalScore（实际得分）

```json
"totalScore": 11.3
```

- **类型**: `float`
- **范围**: 0-满分（如15、25）
- **说明**: 实际得分（非百分比）
- **注意**: 此字段名容易混淆，建议前端使用 percentageScore

---

### 🔷 维度分数（5分制）

#### 3.3 四维度评分体系

```json
{
  "topicScore": 4,        // 主题契合度
  "contentScore": 4,      // 内容完整性
  "structureScore": 4,    // 结构逻辑性
  "languageScore": 3      // 语言准确性
}
```

##### 3.3.1 topicScore - 主题契合度

**评分标准**：
- **5分**：完全切题，立意深刻，观点明确
- **4分**：切题，立意清晰，观点明确
- **3分**：基本切题，立意一般
- **2分**：部分偏离主题
- **1分**：完全离题

**考核要点**：
- 是否理解题目要求
- 是否围绕主题展开
- 观点是否明确
- 立意是否恰当

##### 3.3.2 contentScore - 内容完整性

**评分标准**：
- **5分**：内容丰富，要点齐全，论据充分
- **4分**：内容充实，要点完整，论据较充分
- **3分**：内容基本完整，要点基本覆盖
- **2分**：内容单薄，要点缺失
- **1分**：内容严重不足

**考核要点**：
- 要点是否齐全（如应用文的格式要求）
- 论据是否充分
- 内容是否丰富
- 细节是否到位

##### 3.3.3 structureScore - 结构逻辑性

**评分标准**：
- **5分**：结构严谨，逻辑清晰，层次分明
- **4分**：结构完整，逻辑清晰
- **3分**：结构基本完整，逻辑基本清晰
- **2分**：结构松散，逻辑混乱
- **1分**：无结构或逻辑严重混乱

**考核要点**：
- 开头、正文、结尾是否完整
- 段落划分是否合理
- 逻辑连接是否顺畅
- 过渡词使用是否恰当

##### 3.3.4 languageScore - 语言准确性

**评分标准**：
- **5分**：语言准确，词汇丰富，句式多样
- **4分**：语言基本准确，词汇较丰富，句式有一定变化
- **3分**：语言有少量错误，词汇一般，句式较单一
- **2分**：语言错误较多，影响理解
- **1分**：语言错误严重，难以理解

**考核要点**：
- 语法错误数量
- 拼写错误数量
- 词汇丰富度
- 句式多样性
- 表达准确性

### 🔷 评语与建议

#### 3.4 comment - 总评语

```json
"comment": "这是一篇结构清晰、表达准确的应用文。文章格式正确，要点齐全，语言流畅。建议注意冠词的使用和时态的一致性。优点：1. 开头直截了当，表明写作目的；2. 要点齐全，覆盖了所有要求；3. 结尾得体。不足：1. 存在3处语法错误；2. 时态使用不够统一；3. 部分表达可以更地道。"
```

- **类型**: `string`
- **长度**: 100-500字
- **结构**: 三段式（优点 + 不足 + 总结）
- **要求**:
  - 客观公正
  - 具体明确
  - 有理有据
  - 鼓励性为主

**推荐结构**：
```
第1段：总体评价（2-3句话）
第2段：具体优点（2-3点，每点1-2句话）
第3段：改进建议（2-3点，每点1-2句话）
第4段：总结鼓励（1句话）
```

#### 3.5 suggestion - 改进建议

```json
"suggestion": "建议：1. 注意冠词a/an的使用规则：元音音素前用an，辅音音素前用a；2. 统一时态：全文应保持一致的时态，避免混用；3. 增加过渡词：使用however、therefore、furthermore等连接词使文章更连贯；4. 丰富句式：尝试使用定语从句、状语从句等复杂句式。"
```

- **类型**: `string`
- **长度**: 50-300字
- **格式**: 编号列表或分点陈述
- **要求**:
  - 具体可操作
  - 针对性强
  - 优先级排序
  - 3-5条建议

### 🔷 语法错误标注

#### 3.6 error_set - 错误集合（兼容AYX）

```json
"error_set": [
  {
    "offset": 31,
    "length": 1,
    "message": "语法错误：此处应使用'an'而不是'a'，因为后面单词'good'以元音音素开头",
    "rule": {
      "id": "ARTICLE_A_VS_AN",
      "category": "ERROR_GRAMMAR"
    }
  },
  {
    "offset": 85,
    "length": 3,
    "message": "拼写错误：'recieving'应改为'receiving'（receive+ing，去e加ing）",
    "rule": {
      "id": "SPELLING_ERROR",
      "category": "ERROR_SPELLING"
    }
  }
]
```

##### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| **offset** | int | ✅ | 错误起始位置（字符索引，从0开始） |
| **length** | int | ✅ | 错误字符数 |
| **message** | string | ✅ | 错误描述和修改建议 |
| **rule.id** | string | ✅ | 错误唯一标识 |
| **rule.category** | string | ✅ | 错误分类 |

##### 错误分类（rule.category）

```json
{
  "ERROR_SPELLING": "拼写错误",
  "ERROR_GRAMMAR": "语法错误",
  "ERROR_VERB": "动词错误",
  "ERROR_NOUN": "名词错误",
  "ERROR_ARTICLE": "冠词错误",
  "ERROR_PREPOSITION": "介词错误",
  "ERROR_CONJUNCTION": "连词错误",
  "ERROR_TENSE": "时态错误",
  "COLLOCATIONS": "搭配不当",
  "ERROR_CAPITALIZATION": "大小写错误",
  "ERROR_PUNCTUATION": "标点错误"
}
```

##### message 字段格式规范

```
格式：[错误类型]：[具体错误] + [修改建议] + [原因解释]

示例：
✅ "语法错误：此处应使用'an'而不是'a'，因为后面单词'good'以元音音素开头"
✅ "拼写错误：'recieving'应改为'receiving'（receive+ing，去e加ing）"
✅ "时态错误：全文应使用一般现在时，此处过去时态与全文不一致"

❌ "错误"（太简短）
❌ "Use 'an' instead of 'a'"（没有原因）
```

### 🔷 错误纠正详情（增强版）

#### 3.7 errorCorrection - 详细错误纠正

```json
"errorCorrection": [
  {
    "ori": "【【I hav a good news for you】】",
    "correction": "I have good news for you",
    "reason": "语法错误：1. 'hav'是拼写错误，应改为'have'；2. 'news'是不可数名词，不能用冠词'a'修饰，直接说'good news'即可"
  },
  {
    "ori": "【【I am recieving your letter】】",
    "correction": "I am receiving your letter",
    "reason": "拼写错误：'recieve'是常见拼写错误，正确拼写为'receive'（可用助记：i before e except after c）"
  }
]
```

##### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| **ori** | string | ✅ | 原文（用【【】】包裹以便提取） |
| **correction** | string | ✅ | 修改后的正确句子 |
| **reason** | string | ✅ | 错误原因和修改理由 |

##### ori 字段的特殊标记

使用 **【【】】** 包裹原文的原因：
1. 便于后端从大模型输出中提取原文
2. 避免与正文中的双引号混淆
3. 统一格式标准

**示例**：
```
原文：I hav a good news
ori格式：【【I hav a good news】】
```

##### reason 字段的要求

- 说明错误类型
- 解释错误原因
- 提供修改建议
- 必要时提供语法规则或记忆技巧

**示例**：
```
✅ "拼写错误：'recieve'应改为'receive'（助记：i before e except after c）"
✅ "语法错误：不可数名词'news'前不能加冠词'a'或'an'"
✅ "时态错误：全文叙述现在的事情，应使用一般现在时，保持时态一致"
```

### 🔷 优秀表达详情（增强版）

#### 3.9 expression - 优秀表达点评

```json
"expression": [
  "1. \"express my sincere gratitude\" - 表达地道，语气得体，'sincere'体现了真诚的态度，适合正式申请信等场合使用。",

```

```
3. 用" - "分隔原文和点评
4. 点评要具体，说明好在哪里
5. 每条表达点评30-100字
6. 推荐3-5条
```

##### 点评维度

| 维度 | 说明 | 示例 |

| **表达得体性** | 语气恰当、符合语境 | "适合正式申请信的场合" |

```javascript
// 渲染 error_set
function renderErrors(postText, errorSet) {
  let html = postText;

  // 按offset倒序排列（避免位置偏移）
  const sortedErrors = errorSet.sort((a, b) => b.offset - a.offset);

  sortedErrors.forEach(error => {
    const errorText = postText.substring(
      error.offset,
      error.offset + error.length
    );

    const before = html.substring(0, error.offset);
    const after = html.substring(error.offset + error.length);

    html = before +
      `<span class="error" data-category="${error.rule.category}"
             title="${error.message}">${errorText}</span>` +
      after;
  });

  return html;
}

// CSS样式
.error {
  background-color: #ffebee;
  border-bottom: 2px wavy #f44336;
  cursor: pointer;
}
.error:hover {
  background-color: #ffcdd2;
}
```

```javascript
// 渲染 highlight
function renderHighlights(postText, highlights) {
  let html = postText;

  // 按offset倒序排列
  const sorted = highlights.sort((a, b) => b[1][0] - a[1][0]);

  sorted.forEach(item => {
    const text = item[0];
    const start = item[1][0];
    const end = item[1][1];

    const before = html.substring(0, start);
    const after = html.substring(end);

    html = before +
      `<span class="highlight" title="优秀表达">${text}</span>` +
      after;
  });

  return html;
}
```

## 八、质量标准

### 🔷 评分准确性

| 维度 | 准确率要求 | 测试方法 |
|------|-----------|---------|
| **总分准确率** | 与人工评分相关系数 > 0.85 | 100篇作文双盲测试 |
| **维度分数** | 与人工评分误差 ≤ 1分 | 抽查验证 |
| **错误检测** | 召回率 > 0.85, 精确率 > 0.90 | 标注数据集测试 |

### 🔷 内容质量标准

#### comment（评语）

- ✅ 必须包含总体评价
- ✅ 必须指出2-3个优点
- ✅ 必须指出2-3个不足
- ✅ 长度在100-500字
- ✅ 语法正确，表达流畅

#### errorCorrection（错误纠正）

- ✅ 所有检测到的错误都必须包含
- ✅ ori字段必须与原文完全一致
- ✅ correction必须语法正确
- ✅ reason必须清晰易懂

#### expression（优秀表达）

- ✅ 推荐3-5条
- ✅ 每条点评30-100字
- ✅ 确实是优秀表达（不是普通句子）
- ✅ 点评有理有据

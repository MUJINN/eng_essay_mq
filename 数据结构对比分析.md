## 一、输入数据结构（MQ消息）

### 🔷 通用输入结构（英语 & 语文通用）

```json
{
  "subjectId": 9768863,          // 科目ID（必需）
  "blockId": 23576240,           // 题目块ID（必需）
  "taskKey": "9768863-23576240-1764578529021",  // 任务唯一标识（必需）
  "grade": "高二",               // 年级（必需）
  "totalScore": 60,              // 总分（必需，如60分、15分）
  "model": "ayx",                // 批改模型：ayx/dianxin/dify/n8n（可选，默认dianxin）
  "task": "en-app",              // 任务类型：en-app=英语 / cn=语文（可选，默认en-app）
  "problem": "作文题目...",       // 作文题目要求（可选，英文批改推荐）
  "key": "5.33",                 // 回传标识（可选）

  "students": [                  // 学生作文列表（必需）
    {
      "uploadKey": "82881-112825-10-241",  // 学生唯一标识（必需）
      "post_text": "学生作文内容..."        // 作文正文（必需）
    }
  ]
}
```

### 🔷 英语作文特有字段

```json
{
  "task": "en-app",              // 固定值：英语作文批改
  "problem": "假定你是李华，写一封英文申请信...",  // 英文题目（推荐提供）
  "totalScore": 15               // 通常是15分或25分
}
```

**学生作文示例**：
```json
{
  "uploadKey": "test1",
  "post_text": "I'm Li Hua. I notice that some students eat snacks at mealtimes instead of having meals. Now I'd like to call on you to form a healthy eating habit..."
}
```

### 🔷 语文作文特有字段

```json
{
  "task": "cn",                  // 固定值：语文作文批改
  "problem": "阅读下面的材料，根据要求写作...",  // 语文材料（必需）
  "totalScore": 60               // 通常是60分
}
```

**学生作文示例**：
```json
{
  "uploadKey": "82881-112825-10-241",
  "post_text": "破除人生壁垒,成就独特自我\n\n敬爱的老师和同学们:\n\n大家好!今天我要演讲的的题目是"破除人生壁垒,成就独特自我"..."
}
```

      // 详细批改信息

### 🔷 英语作文输出结果详解

#### 1️⃣ 使用 ayx 模型（自研模型）

```json
{
  "students": [
    {
      "uploadKey": "test1",
      "score": 0.723,                     // 得分率 72.3%
      "highlight": [                      // 亮点表达（词汇、句式）
        ["excellent vocabulary", [10, 30]],
        ["good sentence structure", [50, 75]]
      ],
      "error_set": [                      // 语法错误列表
        {
          "offset": 45,                   // 错误位置（字符偏移量）
          "length": 6,                    // 错误长度
          "message": "Spelling error",    // 错误描述
          "rule": {                       // 错误规则
            "id": "SPELLING_ERROR",
            "category": "ERROR_SPELLING"
          }
        }
      ],
      "comment": "",                      // 总评语（自研模型可能为空）
      "suggestion": [],                   // 建议列表
      "improveArticle": "",               // 升格范文（空）
      "topicScore": 0,                    // 维度分数（自研模型为0）
      "contentScore": 0,
      "structureScore": 0,
      "languageScore": 0,
      "errorCorrection": [],              // 错误纠正详情（空）
      "expression": []                    // 优秀表达（空）
    }
  ]
}
```

**特点**：
- ✅ **有 highlight**：亮点词汇和句式
- ✅ **有 error_set**：详细的语法错误标注（GEC结果）
- ❌ **无维度分数**：topicScore等都是0
- ❌ **无评语**：comment通常为空

#### 2️⃣ 使用 dianxin 模型（电信大模型）

```json
{
  "students": [
    {
      "uploadKey": "test1",
      "score": 0.82,                      // 得分率 82%
      "highlight": [                      // 优秀句子
        ["excellent expression here", [10, 35]]
      ],
      "error_set": [                      // 错误标注
        {
          "offset": 45,
          "length": 6,
          "message": "should be 'receive' not 'recieve'"
        }
      ],
      "comment": "这是一篇结构清晰、表达准确的应用文。文章格式正确，要点齐全，语言流畅。建议注意拼写和时态的准确性。",
      "suggestion": "建议：1. 注意拼写；2. 统一时态",
      "improveArticle": "",               // 升格范文
      "topicScore": 4,                    // 维度分数（5分制）
      "contentScore": 4,
      "structureScore": 5,
      "languageScore": 4,
      "errorCorrection": [                // 错误纠正详情
        {
          "ori": "\"I'm recieving...\"",
          "reason": "语法错误，应修改为\"I'm receiving...\""
        }
      ],
      "expression": [                     // 优秀表达
        "1. \"excellent vocabulary here\" - 用词准确",
        "2. \"good sentence structure\" - 句式丰富"
      ]
    }
  ]
}
```

**特点**：
- ✅ **有完整维度分数**：topicScore, contentScore等
- ✅ **有详细评语**：comment字段
- ✅ **有错误纠正**：errorCorrection数组
- ✅ **有优秀表达**：expression数组

      "suggestion": "建议：1. 充实论据；2. 注意字数要求；3. 提升语言精准度",
      "improveArticle": "（升格后的范文内容）",
      "topicScore": 3,                    // 维度分数（5分制）
      "contentScore": 3,
      "structureScore": 3,
      "languageScore": 3,

- ✅ **完整的维度分数**：基于4个维度评价

- ✅ **升格范文**：improveArticle字段包含改写后的优秀范文
- ✅ **丰富的错误纠正**：errorCorrection包含原文、修改建议
- ✅ **优秀表达提取**：expression包含亮点句子和点评

## 三、核心差异对比

### 📊 输入数据差异

| 字段 | 英语作文 | 语文作文 | 说明 |
|------|---------|---------|------|
| **task** | `"en-app"` | `"cn"` | 任务类型区分 |
| **totalScore** | 通常15/25分 | 通常60分 | 总分不同 |
| **problem** | 英文题目（可选） | 材料作文（必需） | 题目格式不同 |
| **post_text** | 英文内容 | 中文内容 | 作文正文语言 |
| **grade** | 高一/高二/高三 | 相同 | 年级信息通用 |

### 📊 输出数据差异

| 字段 | 英语（ayx模型） | 英语（dianxin模型） | 语文（Dify模型） |
|------|----------------|-------------------|-----------------|
| **score** | ✅ 0-1之间的小数 | ✅ 0-1之间的小数 | ✅ 0-1之间的小数 |
| **highlight** | ✅ 有（词汇/句式） | ✅ 有（优秀句子） | ✅ 有（亮点句子） |
| **error_set** | ✅ 有（GEC语法错误） | ✅ 有（语法错误） | ✅ 有（语言错误） |
| **comment** | ❌ 通常为空 | ✅ 详细评语 | ✅ 详细评语 |
| **suggestion** | ✅ 数组或字符串 | ✅ 文本建议 | ✅ 文本建议 |
| **improveArticle** | ❌ 空 | ❌ 空 | ✅ 升格范文 |
| **topicScore** | ❌ 0 | ✅ 1-5分 | ✅ 1-5分 |
| **contentScore** | ❌ 0 | ✅ 1-5分 | ✅ 1-5分 |
| **structureScore** | ❌ 0 | ✅ 1-5分 | ✅ 1-5分 |
| **languageScore** | ❌ 0 | ✅ 1-5分 | ✅ 1-5分 |
| **errorCorrection** | ❌ 空 | ✅ 详细错误说明 | ✅ 详细错误说明 |
| **expression** | ❌ 空 | ✅ 优秀表达列表 | ✅ 优秀表达列表 |

### 📊 模型选择差异

| 模型 | 适用场景 | 优势 | 劣势 |
|------|---------|------|------|
| **ayx** | 英语作文（自研） | 响应快、成本高 | 无维度分数、无评语 |
| **dianxin** | 英语作文（电信） | 评价全面、维度详细 | 成本较高、依赖外部 |
| **Dify** | 语文作文 | 评价深入、有升格范文 | 仅支持语文 |
| **n8n** | 英语作文（自建） | 可定制、灵活 | 需自行搭建 |

## 四、处理流程对比

### 🔷 英语作文处理流程

```
MQ消息
  ↓
parser_one_essay()
  ↓
┌─────────────────────────────────────────┐
│ model == 'ayx' && task == 'en-app'?     │
└─────────────────────────────────────────┘
  ↓ Yes                    ↓ No
_parser_one_essay()     根据model选择
(自研GEC+AES)            ↓
  ↓                  ├─ ayx → _correct_essay_by_dify()
返回结果               ├─ dianxin → _correct_essay_by_dianxin()
                      └─ n8n → _correct_essay_by_n8n()
  ↓
返回给MQ
```

**关键代码**：
- `model='ayx' && task='en-app'` → 使用自研模型（[english_write.py:479](english_write.py#L479)）
- `model='dianxin'` → 使用电信大模型
- `model='ayx' && task='cn'` → 使用Dify（语文）

### 🔷 语文作文处理流程

```
MQ消息
  ↓
parser_one_essay()
  ↓
model == 'ayx'
  ↓
_correct_essay_by_dify()
  ↓
调用 Dify 工作流 API
  ↓
解析返回结果（维度分数转换）
  ↓
返回给MQ
```

**关键代码**：
- `_correct_essay_by_dify()` → [english_write.py:278](english_write.py#L278)
- 维度分数转换：4分制 → 5分制（[english_write.py:361](english_write.py#L361)）

## 八、总结

### 核心差异总结

1. **任务类型区分**：`task`字段（`en-app` vs `cn`）
2. **总分差异**：英语通常15分，语文60分
3. **返回内容差异**：
   - 英语（ayx）：只有分数和语法错误
   - 英语（dianxin）：完整的维度分数和评语
   - 语文（Dify）：最完整，包含升格范文
4. **模型选择策略**：
   - `model='ayx' && task='en-app'` → 自研英语模型
   - `model='ayx' && task='cn'` → Dify语文模型
   - `model='dianxin'` → 电信英语模型

### 数据完整性

✅ **输入数据**：英语和语文使用统一的MQ格式
✅ **输出数据**：返回格式统一，差异在字段内容
✅ **兼容性**：新旧模型并存，通过model字段选择
